나만의 AI 어시스턴트 만들기

GPTs, Gems, Projects — 매번 긴 설명 없이도 내 스타일을 아는 AI를 직접 설계해 봐요.

프롬프트를 잘 쓰는 것도 중요하지만, 매번 같은 설명을 반복하는 건 번거롭죠. "나는 마케팅 담당자고, 우리 회사 톤앤매너는 친근하고 간결해야 해, 타겟은 20~30대야…" 이런 배경을 매번 입력하지 않아도 되는 방법이 있습니다.

ChatGPT의 GPTs, Gemini의 Gems, Claude의 Projects — 세 AI 모두 "나만의 전용 AI"를 직접 설계할 수 있는 기능을 제공합니다. 한 번 설정해두면 매번 처음부터 설명할 필요 없이, 내 업무에 딱 맞게 작동합니다.

Gemini를 주로 쓴다면 Gems, Claude를 주로 쓴다면 Projects가 가장 빠릅니다. GPTs는 ChatGPT Plus 구독이 필요하지만 타인과 공유할 수 있어서 팀 배포에 유용합니다.

활용 사례 ① — 프롬프트 생성기 만들기

"AI한테 뭘 시키고 싶은데 어떻게 말해야 할지 모르겠어요"라는 분들을 위한 최고의 해법입니다. 프롬프트를 대신 만들어주는 AI를 하나 만들어두면, 앞으로는 AI에게 막연하게 물어봐도 잘 구조화된 프롬프트를 자동으로 생성해 줍니다.

프롬프트 생성기 설정 방법

Gems 또는 Projects를 열고, 아래 내용을 지시사항(Instructions)에 붙여넣으세요.

📋 프롬프트 생성기 지시사항 (복사해서 사용)
당신은 전문 프롬프트 작성 도우미입니다.

사용자가 원하는 작업을 설명하면, 다음 단계로 진행하세요:

1. 목적 파악: 이 프롬프트로 무엇을 얻고 싶은지 1~2가지 핵심 질문으로 확인합니다.
2. 역할 설정: AI에게 맡길 역할을 명확히 정의합니다 (예: "당신은 10년 경력의 카피라이터입니다").
3. 맥락 추가: 대상, 목적, 톤앤매너, 분량 등을 구체화합니다.
4. 출력 형식: 결과물의 형태(목록, 표, 단락 등)를 지정합니다.
5. 완성된 프롬프트를 코드 블록으로 제시합니다.

항상 한국어로 답하고, 전문 용어 대신 쉬운 표현을 사용하세요.

실제 사용 예시

내가 입력한 것

"신제품 출시 이메일 써줘"

생성기가 만들어준 프롬프트

"당신은 B2B SaaS 마케팅 전문가입니다. [제품명]의 신규 출시를 알리는 이메일을 작성해 주세요. 대상: 기존 고객사 담당자. 톤: 전문적이지만 따뜻한. 구성: 제목 + 본문(3단락) + CTA 버튼 문구. 분량: 200자 이내."

💡 이렇게 활용해 보세요
프롬프트 생성기를 Gem/Project로 만들어두면 팀원들에게 공유할 수도 있습니다. "이 AI한테 하고 싶은 거 말하면 프롬프트 만들어줘"라고 안내하면, AI 초보자도 쉽게 좋은 프롬프트를 쓸 수 있습니다.

활용 사례 ② — 업무 전용 AI 어시스턴트

한 번 설정해두면 매번 배경 설명 없이 바로 업무 지시만 내릴 수 있는 나만의 AI를 만들 수 있습니다.

📣
마케팅 카피 도우미
회사 톤앤매너, 타겟 고객, 금지어 등을 미리 등록해두면 매번 설명 불필요
📊
보고서 작성 도우미
회사 보고서 양식, 결재 라인, 문체 스타일을 지정해 일관된 보고서 생성
🙋
고객 응대 도우미
FAQ 문서, 응대 가이드라인을 업로드해두면 일관된 답변 초안 생성
📅
기획안 도우미
과거 기획서 양식, 승인 기준을 등록해 신규 기획안의 구조화 지원

지시사항 작성 5단계

좋은 AI 어시스턴트를 만드는 지시사항은 다음 다섯 가지를 포함하면 됩니다.

  1. 역할 정의 — "당신은 우리 회사의 마케팅 카피 전문가입니다."
  2. 맥락 제공 — "회사: OO기업 / 업종: B2B SaaS / 주요 고객: 중소기업 HR 담당자"
  3. 스타일 규칙 — "문체: 친근하지만 전문적 / 길이: 간결하게 / 금지어: 혁신적, 최고의"
  4. 출력 형식 — "결과물은 항상 제목 + 본문 + 핵심 메시지 한 줄 순서로 제시"
  5. 예시 포함 — 잘 만들어진 예시 결과물을 하나 추가하면 일관성이 크게 높아집니다

Claude Projects 심화 — 설계서 활용법

Claude Projects는 특히 복잡한 업무를 지속적으로 협업할 때 강력합니다. 단순한 지시사항 외에도 업무 설계서를 문서로 업로드하면, Claude가 훨씬 더 구체적이고 일관된 방식으로 도와줍니다.

설계서에 담으면 좋은 내용

📋 작업 컨텍스트
  • 이 프로젝트의 목적과 배경
  • 최종 결과물의 모습
  • 함께 쓰는 도구나 파일 형식
  • 절대 하면 안 되는 것 (제약사항)
🔄 진행 방식
  • 작업 순서 (1단계, 2단계…)
  • 판단이 필요할 때 기준
  • 내가 확인해야 하는 시점
  • 결과물의 검토 기준

실제 설계서 예시 (간략형)

📄 월간 뉴스레터 제작 프로젝트 설계서
목적: 매월 고객사에게 발송하는 뉴스레터 초안 작성
결과물: 제목 3개 안 + 본문(500자) + 마무리 CTA
스타일: 친근한 문체, 전문 용어 최소화, 이모지 1~2개
금지사항: 경쟁사 언급, 가격 정보 포함
작업 순서:
1. 이번 달 주제 키워드 확인
2. 제목 3개 초안 제시 → 내가 선택
3. 선택된 제목으로 본문 작성
4. 검토 요청 후 최종본 확정
💡 설계서는 처음부터 완벽하지 않아도 됩니다
Claude와 대화하면서 "이번엔 이 부분이 별로였어"라고 피드백을 주면, Claude가 설계서 수정안을 제안해 드립니다. 쓰면서 조금씩 다듬어 가면 됩니다.

한 눈에 보는 시작 가이드

1
플랫폼 선택 — 평소 자주 쓰는 AI (Gemini → Gems / Claude → Projects / ChatGPT Plus → GPTs)
2
용도 결정 — 프롬프트 생성기? 특정 업무 도우미? 목적 하나만 정하세요
3
지시사항 작성 — 역할 + 맥락 + 스타일 규칙 + 출력 형식, 5단계로 작성
4
테스트 & 개선 — 실제로 써보고 아쉬운 부분을 AI에게 말하면 수정안 제안
