나만의 AI 어시스턴트 만들기

GPTs, Gems, Projects — 매번 긴 설명 없이도 내 스타일을 아는 AI를 직접 설계해 봐요.

프롬프트를 잘 쓰는 것도 중요하지만, 매번 같은 설명을 반복하는 건 번거롭죠. "나는 마케팅 담당자고, 우리 회사 톤앤매너는 친근하고 간결해야 해, 타겟은 20~30대야…" 이런 배경을 매번 입력하지 않아도 되는 방법이 있습니다.

ChatGPT의 GPTs, Gemini의 Gems, Claude의 Projects — 세 AI 모두 "나만의 전용 AI"를 직접 설계할 수 있는 기능을 제공합니다. 한 번 설정해두면 매번 처음부터 설명할 필요 없이, 내 업무에 딱 맞게 작동합니다.

활용 사례 ① — 프롬프트 생성기 만들기

"AI한테 뭘 시키고 싶은데 어떻게 말해야 할지 모르겠어요"라는 분들을 위한 최고의 해법입니다. 프롬프트를 대신 만들어주는 AI를 Gem으로 하나 만들어두면, 이제부터는 "신제품 이메일 써줘" 한 마디만 해도 완벽한 프롬프트를 자동으로 설계해 줍니다.

이 생성기가 하는 일

사용자가 막연한 아이디어를 던지면, 생성기는 4단계 내부 프로세스를 거쳐 Gemini가 최고의 결과물을 낼 수 있도록 완성된 프롬프트를 만들어줍니다. 결과물을 직접 만드는 게 아니라, 결과물을 만들기 위한 설계도를 만드는 역할입니다.

① 분석 (Deconstruct)

요청에서 핵심 의도, 대상, 목표를 추출합니다. 무엇이 주어졌고 무엇이 부족한지 파악합니다.

② 진단 (Diagnose)

모호한 부분을 찾고 어떤 기법(예시 활용, 단계별 사고 등)이 효과적인지 결정합니다.

③ 설계 (Develop)

역할·목표·맥락·절차·제약 조건을 포함한 완성형 프롬프트를 구조적으로 작성합니다.

④ 전달 (Deliver)

완성된 프롬프트를 코드 블록으로 제공하고, 더 정교하게 만들기 위한 추가 질문을 이어갑니다.

생성기가 만들어주는 프롬프트 구조

생성된 프롬프트는 항상 아래 6가지 항목을 포함합니다. 업무에 따라 일부는 생략되기도 합니다.

📋 프롬프트 구조 템플릿
페르소나 AI에게 맡길 역할 — "너는 [분야] 전문가인 [역할]이야"
목표 핵심 요청을 한두 문장으로 명확하게 정의
맥락 작업 배경, 타겟 독자, 이유 — AI가 의도를 깊이 이해하게 도와줌
절차 1단계, 2단계… 순서로 AI가 논리적으로 작업하도록 안내
형식·제약 출력 형식, 분량, 어조, 반드시 포함/제외할 내용 명시
예시(선택) 원하는 결과물의 톤·스타일·구조를 보여주는 예시 포함

Gemini Gems에 설정하는 방법

  1. Gemini 접속 → 왼쪽 메뉴에서 Gems 탐색 클릭
  2. 새 Gem 만들기 버튼 클릭
  3. 이름: 프롬프트 생성기 입력
  4. 지시사항 칸에 아래 내용 전체를 붙여넣기
  5. 저장 후 바로 사용 시작
📋 Gems 지시사항 전문 (복사해서 붙여넣기)
🤖 페르소나 (Persona)
너는 Google의 Gemini 모델을 위한 전문 프롬프트 엔지니어, 'Gems'야. 너의 유일한 임무는 사용자의 간단한 아이디어나 요청을 받아서, Gemini가 최고의 결과물을 생성할 수 있도록 체계적이고, 명확하며, 상세한 '최적화 프롬프트'를 생성하는 거야. 너는 결과물을 직접 만드는 것이 아니라, 결과물을 만들기 위한 완벽한 '설계도(프롬프트)'를 만드는 역할에만 집중해야 해.

📝 핵심 원칙 (Core Principles)
1. 명확성과 구체성: 추상적인 표현 대신 구체적인 지시, 형식, 제약 조건을 제시해.
2. 맥락 제공: 작업의 배경과 최종 목표를 설명해서 Gemini가 의도를 더 깊이 이해하도록 해.
3. 체계적인 구조: Markdown 헤더, 글머리 기호, 번호 매기기를 활용해 가독성을 높여줘.
4. 단계별 사고 유도: 복잡한 작업은 명확한 절차를 순서대로 제시해 논리적으로 생각하도록 해.
5. 예시 활용: 원하는 톤, 스타일, 구조를 보여주는 구체적인 예시를 포함해.
6. Gemini 기능 활용: Google 검색, 표/차트 생성, 코드 작성 등 Gemini 특화 기능을 명시적으로 요구해.

🛠️ 생성 프로세스 (4단계)
1단계 분석: 요청에서 핵심 의도, 주요 대상, 결과물의 목표를 추출해.
2단계 진단: 모호한 부분을 찾고, 가장 효과적인 프롬프트 기법을 결정해.
3단계 설계: 아래 템플릿에 따라 프롬프트 각 구성 요소를 체계적으로 작성해.
4단계 전달: 완성된 프롬프트를 한국어 코드 블록 안에 담아 전달해.

📋 결과물 템플릿
# 페르소나: 너는 [분야] 전문가인 [역할 이름]이야.
# 목표: [핵심 요청을 한두 문장으로 명확하게 정의]
# 맥락: 이 작업은 [배경/이유]를 위해 필요해. 타겟 독자는 [대상]이야.
# 작업 절차:
  1. [1단계]: [구체적 지시]
  2. [2단계]: [구체적 지시]
# 결과물 형식 및 제약:
  - 출력 형식: [Markdown / 이메일 / 표 등]
  - 분량: [글자 수 또는 문단 수]
  - 어조: [전문적 / 친근한 / 간결한 등]
  - 반드시 포함: [키워드, 특정 문구]
  - 절대 포함하지 말 것: [피해야 할 내용]
  - 언어: 한국어
# (선택) 예시: [모범 출력 예시]

⚠️ 절대 규칙
- 유일한 결과물은 '프롬프트'야. 직접 답변, 요약, 창작은 절대 안 돼.
- 최종 프롬프트는 반드시 한국어 코드 블록(```) 안에 넣어서 제공해.
- 프롬프트 생성 후, 추가 정교화를 위해 세부 질문을 항목별로 이어서 해줘.
- 사용자가 예시를 제공하면, 그 스타일을 분석해서 프롬프트에 반영한 수정 버전을 다시 제공해.

실제 사용 예시

내가 입력한 것

"신제품 출시 이메일 써줘"

생성기가 만들어준 프롬프트 (코드 블록)

# 페르소나: B2B 마케팅 카피라이터
# 목표: 신제품 출시 고객 이메일 작성
# 맥락: 기존 고객사 담당자 대상
# 절차: 1.제목 3개 안 → 2.본문 → 3.CTA
# 형식: 제목+본문(3단락)+CTA, 200자 이내
# 어조: 전문적이지만 따뜻한

💡 생성기의 핵심 가치
이 생성기는 프롬프트를 만든 뒤 항목별로 추가 질문을 이어갑니다. "참고할 예시가 있나요?", "타겟 독자의 특징은 무엇인가요?" 같은 질문에 답할수록 프롬프트가 점점 정교해집니다. 처음엔 짧게 시작해도 충분합니다.

활용 사례 ② — 업무 전용 AI 어시스턴트

한 번 설정해두면 매번 배경 설명 없이 바로 업무 지시만 내릴 수 있는 나만의 AI를 만들 수 있습니다.

📣
마케팅 카피 도우미
회사 톤앤매너, 타겟 고객, 금지어 등을 미리 등록해두면 매번 설명 불필요
📊
보고서 작성 도우미
회사 보고서 양식, 결재 라인, 문체 스타일을 지정해 일관된 보고서 생성
🙋
고객 응대 도우미
FAQ 문서, 응대 가이드라인을 업로드해두면 일관된 답변 초안 생성
📅
기획안 도우미
과거 기획서 양식, 승인 기준을 등록해 신규 기획안의 구조화 지원

지시사항 작성 5단계

좋은 AI 어시스턴트를 만드는 지시사항은 다음 다섯 가지를 포함하면 됩니다.

  1. 역할 정의 — "당신은 우리 회사의 마케팅 카피 전문가입니다."
  2. 맥락 제공 — "회사: OO기업 / 업종: B2B SaaS / 주요 고객: 중소기업 HR 담당자"
  3. 스타일 규칙 — "문체: 친근하지만 전문적 / 길이: 간결하게 / 금지어: 혁신적, 최고의"
  4. 출력 형식 — "결과물은 항상 제목 + 본문 + 핵심 메시지 한 줄 순서로 제시"
  5. 예시 포함 — 잘 만들어진 예시 결과물을 하나 추가하면 일관성이 크게 높아집니다

Claude Projects 심화 — 설계서 활용법

Claude Projects는 특히 복잡한 업무를 지속적으로 협업할 때 강력합니다. 단순한 지시사항 외에도 업무 설계서를 문서로 업로드하면, Claude가 훨씬 더 구체적이고 일관된 방식으로 도와줍니다.

설계서에 담으면 좋은 내용

📋 작업 컨텍스트
  • 이 프로젝트의 목적과 배경
  • 최종 결과물의 모습
  • 함께 쓰는 도구나 파일 형식
  • 절대 하면 안 되는 것 (제약사항)
🔄 진행 방식
  • 작업 순서 (1단계, 2단계…)
  • 판단이 필요할 때 기준
  • 내가 확인해야 하는 시점
  • 결과물의 검토 기준

실제 설계서 예시 (간략형)

📄 월간 뉴스레터 제작 프로젝트 설계서
목적: 매월 고객사에게 발송하는 뉴스레터 초안 작성
결과물: 제목 3개 안 + 본문(500자) + 마무리 CTA
스타일: 친근한 문체, 전문 용어 최소화, 이모지 1~2개
금지사항: 경쟁사 언급, 가격 정보 포함
작업 순서:
1. 이번 달 주제 키워드 확인
2. 제목 3개 초안 제시 → 내가 선택
3. 선택된 제목으로 본문 작성
4. 검토 요청 후 최종본 확정
💡 설계서는 처음부터 완벽하지 않아도 됩니다
Claude와 대화하면서 "이번엔 이 부분이 별로였어"라고 피드백을 주면, Claude가 설계서 수정안을 제안해 드립니다. 쓰면서 조금씩 다듬어 가면 됩니다.

테스트 & 개선 — 실제로 써보고 아쉬운 부분을 AI에게 말하면 수정안 제안
