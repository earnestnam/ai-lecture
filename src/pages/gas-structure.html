📁 GAS 프로젝트, 어떻게 생겼나요?

clasp로 GAS 프로젝트를 내 컴퓨터에 받으면(clone), 폴더 안에 여러 파일이 생겨요. 처음 보면 뭐가 뭔지 헷갈릴 수 있는데, 구조를 한 번만 이해해두면 이후 개발이 훨씬 수월해집니다.

GAS 프로젝트의 핵심은 두 가지예요. 하나는 프로젝트 설정을 담은 appsscript.json, 나머지는 실제 코드가 들어가는 .gs 확장자 파일들이에요. 마치 레스토랑의 주방처럼, appsscript.json은 주방의 설비·규정 문서이고, .gs 파일들은 실제 요리 레시피라고 생각하면 됩니다.

💡 .gs 파일 = JavaScript 파일: GAS의 코드 파일 확장자는 .gs지만, 내용은 JavaScript와 동일해요. VSCode에서 JS 문법 강조가 적용되도록 설정하면 더 편하게 작업할 수 있어요.

⚙️ appsscript.json 파일 설명

모든 GAS 프로젝트 루트에는 appsscript.json 파일이 있어요. 이 파일은 프로젝트의 기본 설정값을 담고 있습니다. 아래는 웹 앱으로 배포하는 프로젝트의 전형적인 예시예요.

appsscript.json 
{
  "timeZone": "Asia/Seoul",
  "dependencies": {
    "enabledAdvancedServices": []
  },
  "webapp": {
    "executeAs": "USER_DEPLOYING",
    "access": "ANYONE"
  },
  "exceptionLogging": "STACKDRIVER",
  "runtimeVersion": "V8"
}

timeZone

스크립트 실행 시 기준 시간대. 한국 시간을 쓰려면 "Asia/Seoul"로 설정해요.

dependencies

고급 Google 서비스(BigQuery, Drive API 등)를 활성화할 때 여기에 추가해요. 기본은 비어있어요.

webapp

웹 앱으로 배포할 때 사용. executeAs는 실행 주체, access는 접근 권한이에요.

runtimeVersion

JavaScript 엔진 버전. "V8"로 설정하면 최신 JS 문법을 사용할 수 있어요.

🔀 Standalone vs Container-bound 차이

GAS 프로젝트는 만드는 방식에 따라 두 종류로 나뉘어요. 어떤 걸 선택하느냐에 따라 접근 가능한 서비스와 주요 용도가 달라지니, 미리 알아두면 좋아요.

구분 Standalone Container-bound
생성 방법 script.google.com에서 직접 생성 Sheets·Docs·Forms 등에서 "확장 프로그램 → Apps Script"
연결된 서비스 특정 파일에 묶이지 않음. 다양한 Google 서비스에 접근 가능 연결된 파일(Sheets 등)에 간편하게 접근 가능
주요 용도 웹 앱, API 서버, 독립 자동화 스크립트 특정 시트 데이터 처리, 폼 제출 시 자동화
권장 상황 여러 팀이 공유하는 도구, 외부에 URL로 공개하는 서비스 특정 스프레드시트와 1:1로 연결되는 단순 자동화

🗂️ 파일 분리 전략

코드가 길어지면 하나의 파일에 모든 내용을 넣기보다, 역할에 따라 파일을 나누는 게 훨씬 관리하기 쉬워요. 아래 구조를 기본 템플릿으로 사용해보세요.

📄 Code.gs

메인 진입점. 외부에서 호출되는 핵심 함수들을 모아두는 곳이에요.

예: doGet(), doPost(), main()

🔧 Utils.gs

공통 유틸리티 함수 모음. 여러 곳에서 반복해서 쓰는 기능을 여기에 모아요.

예: 날짜 포맷, 이메일 발송, 로그 기록

⚙️ Config.gs

상수와 설정값 모음. 자주 바뀌는 값은 여기서 한 번에 관리해요.

예: 시트 이름, 이메일 주소, 열 번호

💻 clasp clone 실습

이미 Google Drive에 있는 GAS 프로젝트를 로컬로 가져오려면 clasp clone 명령어를 사용해요.

1) scriptId 찾기:

  1. GAS 에디터를 열어요 (script.google.com 또는 Sheets → 확장 프로그램 → Apps Script)
  2. 왼쪽 메뉴에서 ⚙️ 프로젝트 설정(Project Settings)을 클릭해요
  3. "스크립트 ID" 항목에 표시된 긴 문자열을 복사해요

2) clasp clone 실행:

터미널 
# 원하는 폴더로 이동한 후 실행
clasp clone [여기에_scriptId_붙여넣기]

# 예시
clasp clone 1BxAbCdEfGhIjKlMnOpQrStUvWxYz_abcdefghij123456

실행하면 현재 폴더에 프로젝트 파일들이 다운로드돼요. appsscript.json과 .gs 파일들이 보이면 성공입니다.

⚠️ 빈 폴더에서 실행하세요: clasp clone은 현재 폴더에 파일을 만들어요. 기존 파일이 있는 폴더에서 실행하면 충돌이 날 수 있으니, 새 폴더를 만들고 그 안에서 실행하는 걸 권장해요.
