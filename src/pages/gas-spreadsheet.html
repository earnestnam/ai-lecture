📊 스프레드시트를 코드로 다루는 법

Google Sheets를 마우스로 클릭하고 타이핑하는 건 누구나 할 수 있어요. 그런데 코드로 다루면, 수천 개의 행을 0.1초 만에 처리하거나, 조건에 맞는 행만 골라 이메일을 보내거나, 매일 자동으로 데이터를 업데이트하는 것도 가능해집니다.

GAS에서 스프레드시트를 다루는 핵심 객체는 SpreadsheetApp이에요. 마치 시트를 다루는 리모컨 같은 거예요. 이 리모컨에서 시작해서, 원하는 시트 → 원하는 셀 범위 → 데이터 읽기/쓰기 순으로 접근합니다.

💡 SpreadsheetApp은 GAS 내장 객체예요: 별도로 import하거나 설치할 필요 없이, GAS 코드 어디서든 SpreadsheetApp을 바로 사용할 수 있어요. Google이 제공하는 기본 도구입니다.

🔑 SpreadsheetApp 기본 패턴

데이터를 읽거나 쓰려면 항상 이 순서를 따라요: 스프레드시트 → 시트 → 범위 → 값

Code.gs — 기본 접근 패턴 
// 1단계: 현재 스크립트와 연결된 스프레드시트를 가져와요
const ss = SpreadsheetApp.getActiveSpreadsheet();

// 2단계: 원하는 시트를 이름으로 가져와요
const sheet = ss.getSheetByName('신청자 명단');

// 3단계: 원하는 셀 범위를 지정해요
//   getRange(시작행, 시작열, 행수, 열수) 순서예요
const range = sheet.getRange(2, 1, 10, 4); // 2행 1열부터 10행 4열까지

// 4단계-A: 셀 하나의 값 읽기
const singleCell = sheet.getRange(2, 1); // B행 A열
const value = singleCell.getValue();     // 문자열 또는 숫자 하나 반환

// 4단계-B: 범위 전체 값 읽기 (2차원 배열로 반환)
const values = range.getValues(); // [[행1열1, 행1열2, ...], [행2열1, ...], ...]

// 4단계-C: 셀에 값 쓰기
singleCell.setValue('홍길동');

// 4단계-D: 범위에 값 쓰기 (2차원 배열로 전달)
range.setValues([
  ['홍길동', 'hong@example.com', '서울', '완료'],
  ['김영희', 'kim@example.com',  '부산', '대기'],
]);

🔲 getValues() / setValues() — 2차원 배열 이해하기

getValues()가 반환하는 2차원 배열이 처음엔 헷갈릴 수 있어요. 스프레드시트의 표 구조가 그대로 배열로 변환된다고 생각하면 쉬워요.

  A열 (이름) B열 (이메일) C열 (상태)
2행 홍길동 hong@example.com 완료
3행 김영희 kim@example.com 대기
4행 이민준 lee@example.com 완료

위 표를 getValues()로 읽으면 아래와 같은 배열이 돼요:

Code.gs — 2차원 배열 읽기 및 쓰기 
const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('신청자 명단');

// 2행부터 4행까지, A~C열 데이터 읽기
const data = sheet.getRange(2, 1, 3, 3).getValues();

// data의 구조:
// [
//   ['홍길동', 'hong@example.com', '완료'],  // 첫 번째 행 (인덱스 0)
//   ['김영희', 'kim@example.com',  '대기'],  // 두 번째 행 (인덱스 1)
//   ['이민준', 'lee@example.com',  '완료'],  // 세 번째 행 (인덱스 2)
// ]

// 특정 값에 접근하기: data[행 인덱스][열 인덱스]
const firstName  = data[0][0]; // '홍길동'
const firstEmail = data[0][1]; // 'hong@example.com'
const thirdStatus = data[2][2]; // '완료'

// 수정된 데이터를 다시 시트에 쓰기
const updatedData = [
  ['홍길동', 'hong@example.com', '완료'],
  ['김영희', 'kim@example.com',  '완료'], // 상태 업데이트
  ['이민준', 'lee@example.com',  '완료'],
];
sheet.getRange(2, 1, 3, 3).setValues(updatedData);

🔍 데이터 필터링, 특정 행 찾기 패턴

실제 업무에서는 전체 데이터를 읽은 뒤, 특정 조건(이메일, 상태, 날짜 등)에 맞는 행만 골라내야 할 때가 많아요. JavaScript의 배열 메서드를 활용하면 간결하게 처리할 수 있어요.

Code.gs — 필터링 및 특정 행 찾기 
const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('신청자 명단');

// 헤더를 제외한 전체 데이터 가져오기
// getLastRow(): 데이터가 있는 마지막 행 번호를 자동으로 구해줘요
const lastRow = sheet.getLastRow();
const data = sheet.getRange(2, 1, lastRow - 1, 3).getValues();
// data 구조: [[이름, 이메일, 상태], ...]

// 패턴 1: 특정 이메일을 가진 행 찾기 (find)
const targetEmail = 'hong@example.com';
const foundRow = data.find(row => row[1] === targetEmail);
// foundRow: ['홍길동', 'hong@example.com', '완료'] (없으면 undefined)

if (foundRow) {
  Logger.log(`이름: ${foundRow[0]}, 상태: ${foundRow[2]}`);
}

// 패턴 2: 상태가 '대기'인 행만 골라내기 (filter)
const pendingRows = data.filter(row => row[2] === '대기');
// pendingRows: [['김영희', 'kim@example.com', '대기'], ...]

// 패턴 3: 대기 상태인 사람들에게 이메일 보내기
pendingRows.forEach(row => {
  const name  = row[0];
  const email = row[1];
  GmailApp.sendEmail(
    email,
    `[안내] ${name}님, 신청이 확인 중입니다`,
    `${name}님, 현재 신청이 검토 중이에요. 처리 완료 시 다시 안내드릴게요.`
  );
});

🤖 Claude Code로 스프레드시트 함수 생성하기

패턴을 외울 필요 없어요. Claude Code에게 원하는 기능을 설명하면 코드를 바로 만들어줍니다. 아래는 효과적인 프롬프트 예시와 예상 출력이에요.

Claude Code에게 이렇게 요청해보세요:
Google Apps Script 코드를 작성해줘.

시트 이름은 '교육 신청자'이고, 구조는 이래:
A열: 이름, B열: 이메일, C열: 부서, D열: 신청 상태 ('대기' 또는 '완료')
헤더는 1행이고 데이터는 2행부터 시작해.

기능: 신청 상태가 '대기'인 행을 모두 찾아서,
해당 사람의 이메일로 "신청이 접수되었습니다"라는 확인 메일을 보내고,
D열 상태를 '완료'로 업데이트해줘.

완전히 복사해서 바로 쓸 수 있는 코드로 줘.

위 프롬프트에 대해 Claude Code가 생성할 코드는 아래와 비슷한 형태가 돼요:

Code.gs — Claude Code 생성 예시 출력 
function sendConfirmationEmails() {
  const ss    = SpreadsheetApp.getActiveSpreadsheet();
  const sheet = ss.getSheetByName('교육 신청자');

  if (!sheet) {
    Logger.log('시트를 찾을 수 없습니다: 교육 신청자');
    return;
  }

  const lastRow = sheet.getLastRow();
  if (lastRow < 2) {
    Logger.log('처리할 데이터가 없습니다.');
    return;
  }

  // 2행부터 전체 데이터 읽기 (A~D열)
  const data = sheet.getRange(2, 1, lastRow - 1, 4).getValues();

  data.forEach((row, index) => {
    const name   = row[0]; // A열: 이름
    const email  = row[1]; // B열: 이메일
    const status = row[3]; // D열: 신청 상태

    // 상태가 '대기'인 경우에만 처리
    if (status === '대기') {
      // 확인 이메일 발송
      GmailApp.sendEmail(
        email,
        '[교육 신청] 신청이 접수되었습니다',
        `${name}님, 안녕하세요.\n교육 신청이 정상적으로 접수되었습니다.\n감사합니다.`
      );

      // D열 상태를 '완료'로 업데이트
      // index는 0부터 시작하므로 실제 행 번호는 index + 2
      sheet.getRange(index + 2, 4).setValue('완료');
      Logger.log(`처리 완료: ${name} (${email})`);
    }
  });

  Logger.log('모든 대기 건 처리 완료!');
}
💡 성능 팁 — getValues() 한 번이 getValue() 여러 번보다 빨라요: 셀을 하나씩 읽는 getValue()를 루프 안에서 반복 호출하면 네트워크 통신이 그만큼 반복돼서 매우 느려져요. 항상 getValues()로 범위 전체를 한 번에 읽은 뒤 JavaScript에서 처리하세요. 100행 기준으로 속도가 10배 이상 차이날 수 있어요.
⚠️ setValues() 배열 크기를 정확히 맞추세요: setValues()에 전달하는 2차원 배열의 행·열 수가 getRange()에서 지정한 범위와 다르면 오류가 나요. 배열 크기와 범위를 항상 일치시켜주세요.
