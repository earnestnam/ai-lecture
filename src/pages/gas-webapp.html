웹앱 서비스 배포

⏱ 45분 중급

doGet/doPost로 HTML 서비스를 만들고 URL로 배포하는 전 과정을 실습해요.

🌐 GAS로 웹 서비스를 만든다?

Google Apps Script는 단순 자동화 도구를 넘어, URL로 접근 가능한 웹 페이지를 직접 서빙할 수 있어요. 별도의 서버나 호스팅 비용 없이, 구글 인프라 위에서 팀 내부 도구를 뚝딱 만들어 배포할 수 있다는 뜻이에요.

예를 들어, 교육 신청 폼·행사 등록 페이지·재고 조회 화면을 GAS 웹앱으로 만들면 조직 내 누구나 링크만 클릭하면 바로 사용할 수 있어요. React나 Vue 같은 복잡한 프레임워크 없이도 가능하고, 스프레드시트 DB(이전 페이지에서 배운 CRUD)와 바로 연결됩니다.

💡 GAS 웹앱이 특히 빛나는 경우: 사내 시스템에 별도 계정 만들기 어려울 때, Google 계정으로 접근 제어가 필요할 때, 데이터가 Google Sheets에 있을 때. 외부 고객용보다는 팀 내 내부 도구에 최적이에요.

🔀 doGet() / doPost() 라우팅 구조

GAS 웹앱의 진입점은 두 함수예요. doGet(e)는 브라우저에서 URL을 열 때(GET 요청), doPost(e)는 폼을 제출할 때(POST 요청) 자동으로 호출돼요.

GAS — doGet() 기본 구조 
/**
 * GET 요청 진입점 — 브라우저에서 URL을 열면 이 함수가 실행됩니다.
 * @param {Object} e - 요청 이벤트 객체
 *   e.parameter        : URL 쿼리 파라미터 (?key=value)
 *   e.pathInfo         : URL 경로 추가 정보
 */
function doGet(e) {
  // URL에서 페이지 파라미터 읽기 (?page=list 처럼 사용)
  const page = e.parameter.page || 'index';

  // 페이지별 다른 HTML 파일 서빙
  let template;
  if (page === 'list') {
    template = HtmlService.createTemplateFromFile('list');
  } else {
    template = HtmlService.createTemplateFromFile('index');
  }

  // 데이터를 HTML 템플릿에 주입
  template.applicants = getAllApplicants(); // CRUD 페이지에서 만든 함수

  return template.evaluate()
    .setTitle('교육 신청 시스템')
    .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL);
}
GAS — doPost() 폼 제출 처리 /** * POST 요청 진입점 — HTML 폼 제출 또는 fetch POST 요청 처리 * @param {Object} e - 요청 이벤트 객체 * e.parameter : application/x-www-form-urlencoded 데이터 * e.postData.contents : 요청 바디 문자열 (JSON 등) */ function doPost(e) { try { // 방법 1: HTML 폼 데이터 (e.parameter 사용) const name = e.parameter.name; const email = e.parameter.email; const course = e.parameter.course; // 방법 2: JSON 바디 파싱 (fetch로 보낼 때) // const body = JSON.parse(e.postData.contents); // const { name, email, course } = body; // 스프레드시트에 저장 const id = createApplicant(name, email, course); // 성공 응답 (JSON) const result = { success: true, id: id }; return ContentService .createTextOutput(JSON.stringify(result)) .setMimeType(ContentService.MimeType.JSON); } catch (err) { // 오류 응답 const result = { success: false, error: err.message }; return ContentService .createTextOutput(JSON.stringify(result)) .setMimeType(ContentService.MimeType.JSON); } }

📄 HtmlService — HTML 파일 서빙 패턴

GAS 프로젝트에 HTML 파일을 추가하면, HtmlService.createTemplateFromFile()로 그 파일을 웹 페이지로 서빙할 수 있어요. GAS의 스크립틀릿(Scriptlet) 문법으로 서버 데이터를 HTML에 바로 넣을 수도 있어요.

GAS HTML 파일 — index.html (스크립틀릿 예시) <!DOCTYPE html> <html lang="ko"> <head> <meta charset="UTF-8"> <title>교육 신청</title> <base target="_top"> <!-- GAS 웹앱에서 링크가 제대로 열리려면 필요 --> </head> <body> <!-- 스크립틀릿: GAS 코드 실행 결과를 HTML에 출력 --> <!-- <?= 값 ?> : HTML 이스케이프 후 출력 (XSS 방지, 권장) --> <!-- <? 코드 ?> : 실행만 하고 출력 없음 --> <h1>안녕하세요! 현재 신청자 수: <?= applicants.length ?>명</h1> <ul> <? for (const person of applicants) { ?> <li><?= person.name ?> — <?= person.status ?></li> <? } ?> </ul> <hr> <h2>신청하기</h2> <form id="applyForm"> <input type="text" name="name" placeholder="이름" required> <input type="email" name="email" placeholder="이메일" required> <input type="text" name="course" placeholder="신청 과정" required> <button type="submit">신청</button> </form> <script> // GAS 함수를 HTML에서 호출: google.script.run document.getElementById('applyForm').addEventListener('submit', function(e) { e.preventDefault(); const formData = new FormData(this); const data = Object.fromEntries(formData); // google.script.run으로 GAS 서버 함수 호출 google.script.run .withSuccessHandler(res => alert(`신청 완료! ID: ${res}`)) .withFailureHandler(err => alert(`오류: ${err.message}`)) .createApplicant(data.name, data.email, data.course); }); </script> </body> </html>
💡 google.script.run이란? HTML 파일(클라이언트) 에서 GAS 서버 함수를 호출하는 브릿지예요. .withSuccessHandler()는 성공했을 때, .withFailureHandler()는 오류가 났을 때 실행될 콜백을 지정해요. GAS 웹앱에서만 동작하는 특수 API예요.

🔐 배포 설정 — 액세스 권한 선택

웹앱을 배포할 때 누가 접근할 수 있는지 반드시 설정해야 해요. 잘못 설정하면 외부에 민감한 데이터가 노출될 수 있어요.

🔒 나만 접근 가능

본인 계정으로만 접근 가능. 개발·테스트 중에 사용해요.

Execute as: Me
Who has access: Only myself

🏢 조직 내 공개

같은 Google Workspace 도메인 구성원만 접근 가능. 사내 도구에 적합해요.

Execute as: Me
Who has access: Anyone in [도메인]

🌍 전체 공개

구글 계정 없이도 접근 가능. 외부 공개 폼·도구에 사용하되 보안에 주의하세요.

Execute as: Me
Who has access: Anyone

🚀 Step-by-Step 배포 가이드

코드 작성이 끝났다면 아래 순서대로 배포하세요.

  1. Apps Script 에디터 상단 오른쪽의 "배포" 버튼 클릭
  2. 드롭다운에서 "새 배포" 선택
  3. "유형 선택" → "웹 앱" 클릭
  4. 설명란에 버전 메모 입력 (예: v1.0 — 초기 배포)
  5. Execute as(코드 실행 주체): 스프레드시트 소유자 본인 계정 선택 권장
  6. Who has access(접근 권한): 용도에 맞게 선택 (위 표 참고)
  7. "배포" 버튼 클릭 → 권한 승인 팝업 허용
  8. 웹 앱 URL이 생성됨 → 복사해서 팀원과 공유!

🔄 코드 수정 후 배포 업데이트하는 법

  1. 코드 수정 후 저장 (Ctrl+S)
  2. "배포" → "배포 관리" 클릭
  3. 연필(✏️) 아이콘 클릭 → "새 버전" 선택
  4. 버전 설명 입력 후 "배포" 클릭
  5. 기존 URL은 그대로 유지됩니다 — 별도로 공유할 필요 없어요
💡 개발 중에는 ?dev=true URL 사용: 배포된 URL 뒤에 ?dev=true를 붙이면 저장할 때마다 최신 코드가 바로 반영되는 개발 버전 URL로 접근할 수 있어요. 단, 본인 계정으로만 접근 가능하고 팀원에게 공유하면 안 됩니다. 테스트 후 정식 배포하면 깔끔해요.
⚠️ CORS 및 인증 주의사항: GAS 웹앱은 기본적으로 Google의 인증을 거쳐야 실행돼요. 외부 사이트에서 fetch로 GAS 웹앱 URL을 호출하면 CORS 오류가 날 수 있어요. 외부 API 용도로 쓰려면 doPost()에서 CORS 헤더를 포함한 ContentService 응답을 반환하거나, 접근 권한을 "Anyone (anonymous)"으로 설정해야 해요. 사내 도구라면 이런 복잡한 상황을 피해 조직 내 공개 설정을 권장해요.
