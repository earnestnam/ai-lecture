clasp push·pull·deploy 명령어로 로컬↔GAS 동기화를 관리해요.

🔄 로컬 코드와 GAS를 연결하는 clasp

GAS를 브라우저 에디터에서만 작성하다 보면 한계에 부딪힙니다. 자동완성이 약하고, Git 연동이 안 되고, 여러 파일을 동시에 편집하기도 불편하죠. 그래서 등장한 것이 clasp(Command Line Apps Script)입니다.

clasp는 구글이 공식으로 제공하는 CLI 도구예요. 로컬 VSCode에서 GAS 코드를 작성하고 명령어 한 줄로 GAS 서버에 올릴 수 있습니다. Claude Code와 함께 사용하면 AI가 코드를 생성 → 로컬에서 확인 → clasp push → GAS에서 실행까지 완전한 로컬 개발 환경이 완성됩니다.

clasp가 하는 일을 한 줄로 표현하면: "로컬 파일 ↔ GAS 프로젝트 양방향 동기화"입니다.

⌨️ clasp 주요 명령어

처음에는 명령어가 많아 보이지만, 실제로 매일 쓰는 것은 push와 pull 두 가지예요. 나머지는 필요할 때 찾아 쓰면 됩니다.

명령어 설명 언제 사용하나요?
clasp clone [id] 기존 GAS 프로젝트를 로컬로 복제 이미 GAS에 있는 프로젝트를 로컬에서 관리하기 시작할 때
clasp push 로컬 파일 → GAS 서버 업로드 로컬에서 코드를 수정한 뒤 GAS에 반영할 때 (가장 자주 사용)
clasp pull GAS 서버 → 로컬 파일 다운로드 GAS 에디터에서 직접 수정한 내용을 로컬에 동기화할 때
clasp deploy -d "설명" 현재 코드를 버전으로 배포 웹앱 또는 Add-on으로 릴리즈할 때, 버전 스냅샷을 남길 때
clasp undeploy [id] 배포 버전 삭제 잘못 배포된 버전을 롤백하거나 오래된 버전을 정리할 때
clasp open GAS 에디터를 브라우저에서 열기 실행 로그 확인, 트리거 설정 등 브라우저 작업이 필요할 때
clasp list 내 GAS 프로젝트 목록 조회 어떤 프로젝트가 있는지 확인할 때, clone 전 ID를 찾을 때
clasp logs Stackdriver 로그 스트리밍 트리거로 자동 실행된 함수의 로그를 터미널에서 확인할 때

🚫 .claspignore 파일 활용법

.claspignore는 clasp push 시 GAS 서버에 올리지 않을 파일을 지정하는 파일이에요. Git의 .gitignore와 문법이 동일합니다. 프로젝트 루트에 만들면 됩니다.

GAS 서버에는 실제 스크립트 파일(.gs, .html)만 올리면 됩니다. README, 테스트 파일, 로컬 설정 파일은 제외하세요.

.claspignore 
# 문서 파일 (GAS 서버에 불필요)
README.md
CLAUDE.md
*.md

# 환경 변수 파일 (보안상 절대 업로드 금지)
.env
.env.local

# 테스트 파일
**/*.test.js
**/*.spec.js
tests/

# Node.js 관련 (로컬 개발 도구)
node_modules/
package.json
package-lock.json

# 에디터 설정
.vscode/
.idea/

# 로그 파일
*.log

# OS 파일
.DS_Store
Thumbs.db
💡 .claspignore 문법 팁: .gitignore와 완전히 동일한 glob 패턴을 사용합니다. **/*.test.js는 모든 하위 폴더의 .test.js 파일을 제외, tests/는 tests 폴더 전체를 제외합니다. 단, GAS는 폴더 구조를 지원하지 않으므로 서브폴더의 .gs 파일도 루트로 평탄화(flatten)되어 업로드됩니다.

🏷️ 버전 관리 (clasp deploy)

clasp push와 clasp deploy는 다른 개념입니다. 혼동하기 쉬우니 명확히 구분하세요.

  • clasp push: 코드를 GAS 서버에 동기화. "헤드(HEAD)" 상태를 업데이트. 버전 번호가 없음.
  • clasp deploy: 현재 코드를 스냅샷으로 고정해 버전 번호를 부여. 웹앱이나 트리거에서 특정 버전을 지정해 사용 가능.

실무에서는 개발 중에는 push만 쓰고, 안정화된 버전을 운영 환경에 배포할 때 deploy를 씁니다.

배포 워크플로 명령어 
# 1. 코드 수정 후 GAS에 동기화
clasp push

# 2. 현재 코드를 버전으로 배포 (-d 옵션으로 설명 추가)
clasp deploy -d "v1.2 - 슬랙 알림 기능 추가"

# 3. 배포된 버전 목록 확인
clasp deployments

# 출력 예시:
# AKfycbxA...  @1  - v1.0 초기 배포
# AKfycbxB...  @2  - v1.1 오류 수정
# AKfycbxC...  @3  - v1.2 슬랙 알림 기능 추가

# 4. 특정 버전으로 롤백 (이전 버전 재배포)
clasp deploy -i "AKfycbxB..." -d "v1.1로 롤백"

# 5. 불필요한 배포 삭제
clasp undeploy AKfycbxA...

🚀 로컬 개발 → 배포 사이클

clasp를 이용한 전체 개발 흐름을 단계별로 살펴봅니다. 처음에는 낯설어 보이지만 몇 번 반복하면 손에 익습니다.

1

로컬 개발

VSCode에서 .gs 파일을 수정합니다. Claude Code를 활용해 코드를 작성하거나 수정하세요. 자동완성, 린트, Git 등 로컬 개발 도구를 모두 활용할 수 있습니다.

2

clasp push

터미널에서 clasp push를 실행합니다. 로컬 파일이 GAS 서버에 업로드됩니다. 몇 초면 완료됩니다.

3

GAS 에디터에서 확인

clasp open으로 브라우저의 GAS 에디터를 열어 코드가 올바르게 반영됐는지 확인합니다. 함수를 실행해보고 실행 로그를 확인하세요.

4

오류 수정 반복

오류가 있으면 로그를 복사해 Claude Code에 전달, 수정된 코드를 받아 다시 1번으로 돌아갑니다. 이 사이클이 빠를수록 개발 속도가 올라갑니다.

5

배포 (clasp deploy)

기능이 완성되고 충분히 테스트됐으면 clasp deploy -d "버전 설명"으로 버전을 고정합니다. 웹앱이나 추가기능(Add-on)이라면 이 버전 ID를 공유하세요.

⚠️ .clasp.json을 공개 저장소에 올리지 마세요: .clasp.json 파일에는 스크립트 ID(scriptId)가 포함됩니다. 이 ID가 공개되면 누구나 해당 GAS 프로젝트에 접근을 시도할 수 있습니다. 공개 GitHub 저장소를 사용한다면 반드시 .gitignore에 .clasp.json을 추가하세요. 팀 내 공유가 필요하다면 비공개(private) 저장소를 사용하거나 스크립트 ID를 환경변수로 분리하는 방법을 고려하세요.
💡 clasp push --watch로 자동 동기화: 개발 중에 매번 clasp push를 입력하는 게 번거롭다면 clasp push --watch를 사용해보세요. 파일 변경을 감지해 자동으로 GAS에 업로드해줍니다. 터미널 탭 하나를 이 명령어에 전용으로 두면 저장할 때마다 자동 동기화가 됩니다. 단, 개발 중에만 사용하고 작업이 끝나면 Ctrl+C로 종료하세요.
