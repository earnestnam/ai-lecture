스프레드시트 DB CRUD

⏱ 50분 중급

스프레드시트를 DB로 쓰는 레코드 생성·조회·수정·삭제 패턴을 실습해요.

🗄️ 스프레드시트를 데이터베이스로 쓰는 법

"데이터베이스"라고 하면 MySQL이나 PostgreSQL 같은 전문 도구를 떠올리기 쉬운데요, 사실 Google Sheets도 훌륭한 데이터베이스 역할을 할 수 있어요. 특히 팀 내 소규모 데이터 관리에는 스프레드시트 DB가 딱 맞아요.

별도 서버가 필요 없고, 구글 계정만 있으면 바로 시작할 수 있어요. 데이터를 눈으로 직접 확인·수정할 수도 있고, Apps Script와 조합하면 자동화까지 연결되죠. 다만 동시 접속자가 많거나, 수십만 건 이상의 대용량 데이터라면 전문 DB를 쓰는 게 낫습니다.

✅ 스프레드시트 DB가 잘 맞는 경우

  • 팀 내 신청·등록·관리 데이터
  • 건수가 수천~수만 건 이하
  • 동시 수정이 거의 없는 업무
  • 비개발자도 데이터를 직접 볼 필요가 있을 때

❌ 전문 DB가 필요한 경우

  • 수십만 건 이상 대용량 데이터
  • 여러 명이 동시에 자주 수정하는 경우
  • 복잡한 관계형 쿼리가 필요한 경우
  • 보안이 매우 중요한 민감한 정보

📐 스프레드시트 DB 구조 설계

스프레드시트를 DB로 쓰려면 구조를 먼저 정해두는 게 핵심이에요. 나중에 구조를 바꾸면 기존 코드를 전부 수정해야 하거든요.

기본 규칙

  • 1행은 무조건 헤더(컬럼명)로 고정해요. 절대 삭제하거나 순서를 바꾸면 안 돼요.
  • A열은 ID로 예약해요. 레코드를 고유하게 식별하는 값이에요.
  • 컬럼명은 영문 소문자 + 언더스코어로 통일하면 코드 작성이 편해요. (예: created_at, user_email)
  • 한 시트 = 한 종류의 데이터. 여러 종류를 섞지 마세요.

예시: 신청자 관리 시트 구조

A열: id B열: name C열: email D열: department E열: status F열: created_at
1710000001234 김민준 minjun@company.com 마케팅팀 승인 2026-03-20 09:31
1710000005678 이서연 seoyeon@company.com 기획팀 대기 2026-03-21 14:05
1710000009012 박지훈 jihoon@company.com 운영팀 반려 2026-03-22 10:47

➕ Create — 레코드 생성

새 레코드를 추가할 때는 appendRow() 메서드를 써요. 시트의 마지막 행 다음에 데이터를 자동으로 붙여줘요. ID는 현재 시각의 타임스탬프(밀리초)를 활용해 자동 생성해요.

GAS — 레코드 생성 함수 
/**
 * 신청자 레코드를 시트에 추가합니다.
 * @param {string} name    - 이름
 * @param {string} email   - 이메일
 * @param {string} dept    - 부서
 */
function createApplicant(name, email, dept) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheet = ss.getSheetByName('신청자'); // 시트 이름과 맞춰주세요

  // ID: 현재 시각 밀리초 (중복 없는 고유값)
  const id = new Date().getTime().toString();

  // 등록 시각 문자열로 변환
  const createdAt = Utilities.formatDate(
    new Date(),
    'Asia/Seoul',
    'yyyy-MM-dd HH:mm'
  );

  // 헤더 순서: id, name, email, department, status, created_at
  sheet.appendRow([id, name, email, dept, '대기', createdAt]);

  Logger.log(`레코드 추가 완료 — ID: ${id}`);
  return id; // 생성된 ID 반환
}

🔍 Read — 레코드 조회

조회는 getValues()로 전체 데이터를 한 번에 읽어온 뒤, JavaScript의 filter나 find로 원하는 행을 찾아요. 매번 시트에 개별 접근하는 것보다 한 번에 읽어오는 게 훨씬 빠릅니다.

GAS — 전체 조회 및 조건 검색 
/**
 * 시트의 모든 레코드를 객체 배열로 반환합니다.
 */
function getAllApplicants() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName('신청자');

  // getValues()는 2차원 배열 반환 [[행1값들], [행2값들], ...]
  const data = sheet.getDataRange().getValues();

  // 1행(인덱스 0)은 헤더이므로 제거
  const headers = data[0];  // ['id', 'name', 'email', ...]
  const rows    = data.slice(1);

  // 각 행을 {id, name, email, ...} 객체로 변환
  return rows.map(row => {
    const obj = {};
    headers.forEach((header, i) => {
      obj[header] = row[i];
    });
    return obj;
  });
}

/**
 * ID로 특정 레코드를 찾습니다.
 * @param {string} targetId - 찾을 레코드의 ID
 */
function findApplicantById(targetId) {
  const all = getAllApplicants();
  // find()는 조건에 맞는 첫 번째 요소 반환, 없으면 undefined
  return all.find(record => record.id === targetId) || null;
}

/**
 * 이메일로 레코드를 찾습니다.
 * @param {string} email - 찾을 이메일 주소
 */
function findApplicantByEmail(email) {
  const all = getAllApplicants();
  return all.find(record => record.email === email) || null;
}

/**
 * 특정 상태의 레코드만 필터링합니다.
 * @param {string} status - '대기', '승인', '반려' 중 하나
 */
function filterByStatus(status) {
  const all = getAllApplicants();
  return all.filter(record => record.status === status);
}

✏️ Update — 레코드 수정

수정은 먼저 대상 행의 번호를 찾은 뒤, 해당 셀의 값을 바꿔요. GAS에서 행 번호는 1부터 시작하고, 헤더가 1행이므로 데이터는 2행부터예요.

GAS — 레코드 수정 함수 /** * ID로 레코드를 찾아 status 값을 업데이트합니다. * @param {string} targetId - 수정할 레코드의 ID * @param {string} newStatus - 새로운 상태값 ('승인' | '반려') */ function updateApplicantStatus(targetId, newStatus) { const sheet = SpreadsheetApp.getActiveSpreadsheet() .getSheetByName('신청자'); const data = sheet.getDataRange().getValues(); // A열(인덱스 0)이 id 컬럼 — 1행은 헤더이므로 i=1부터 시작 for (let i = 1; i < data.length; i++) { if (data[i][0].toString() === targetId) { // 시트 행 번호 = 배열 인덱스 + 1 (1-based) const sheetRow = i + 1; // E열(5번째)이 status — getRange(행, 열) sheet.getRange(sheetRow, 5).setValue(newStatus); Logger.log(`ID ${targetId} → 상태 '${newStatus}'으로 변경`); return true; // 수정 성공 } } Logger.log(`ID ${targetId}를 찾을 수 없습니다.`); return false; // 대상 없음 }

🗑️ Delete — 레코드 삭제

삭제에는 두 가지 방법이 있어요. 물리 삭제(행 자체를 제거)와 소프트 삭제(삭제 표시만 해두기)예요. 업무 데이터는 소프트 삭제를 추천해요. 실수로 지워도 복구할 수 있거든요.

GAS — 소프트 삭제 (권장) 
/**
 * 소프트 삭제: status를 '삭제됨'으로 변경합니다.
 * 실제 행은 유지되므로 필요 시 복구 가능해요.
 * @param {string} targetId - 삭제할 레코드의 ID
 */
function softDeleteApplicant(targetId) {
  // status를 '삭제됨'으로 바꾸는 것으로 처리
  return updateApplicantStatus(targetId, '삭제됨');
}
GAS — 물리 삭제 (주의해서 사용) 
/**
 * 물리 삭제: 행 자체를 제거합니다.
 * ⚠️ 아래에서 위로 삭제해야 행 번호 밀림을 방지할 수 있어요.
 * @param {string} targetId - 삭제할 레코드의 ID
 */
function hardDeleteApplicant(targetId) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName('신청자');

  const data = sheet.getDataRange().getValues();

  // 뒤에서 앞으로 순회해야 deleteRow() 후 인덱스가 안 밀려요
  for (let i = data.length - 1; i >= 1; i--) {
    if (data[i][0].toString() === targetId) {
      sheet.deleteRow(i + 1); // 배열 인덱스 → 시트 행 번호
      Logger.log(`ID ${targetId} 행 삭제 완료`);
      return true;
    }
  }
  return false;
}
⚠️ deleteRow() 사용 시 주의: deleteRow()를 호출하면 그 아래 모든 행의 번호가 하나씩 밀려요. 앞에서 뒤로 반복문을 돌다가 삭제하면 이후 행 번호가 틀어집니다. 반드시 뒤에서 앞으로(역순) 순회하거나, 삭제할 행 번호를 먼저 모두 수집한 뒤 역순으로 삭제하세요.

🔑 ID 자동 생성 전략

레코드마다 고유 ID가 있어야 조회·수정·삭제를 정확히 할 수 있어요. GAS에서 가장 간단하면서도 실용적인 방법은 타임스탬프(밀리초) 기반 ID예요.

GAS — ID 생성 방식 비교 // ① 타임스탬프 기반 (가장 간단, 권장) // 현재 시각의 밀리초 → 13자리 숫자 문자열 const id1 = new Date().getTime().toString(); // 예: "1710000001234" // ② UUID 스타일 (더 고유하지만 코드가 길어요) function generateUUID() { return 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace( /[xy]/g, c => { const r = (Math.random() * 16) | 0; return (c === 'x' ? r : (r & 0x3) | 0x8).toString(16); } ); } const id2 = generateUUID(); // 예: "a3f2c1d0-5b7e-4a9f-8c2d-1e3f5a7b9c0d" // ③ 프리픽스 + 타임스탬프 (가독성이 좋아요) const id3 = `APP-${new Date().getTime()}`; // 예: "APP-1710000001234"
💡 배치 처리로 성능 높이기: 여러 행을 한꺼번에 추가할 때는 appendRow()를 반복 호출하지 말고, 데이터를 배열에 모아 sheet.getRange(startRow, 1, data.length, data[0].length).setValues(data)로 한 번에 쓰세요. 시트 API 호출 횟수를 줄여 실행 속도가 크게 빨라져요.

🤖 Claude Code로 CRUD 일괄 생성 요청하기

시트 구조를 미리 정해두었다면, Claude Code에게 CRUD 함수를 한 번에 만들어달라고 할 수 있어요. 컬럼 정보를 명확히 알려줄수록 더 정확한 코드가 나옵니다.

Claude Code 프롬프트 예시 
Google Apps Script로 스프레드시트 CRUD 함수를 만들어줘.

[시트 정보]
- 시트 이름: "교육신청"
- 스프레드시트는 SpreadsheetApp.getActiveSpreadsheet()로 가져와

[컬럼 구조 — 1행이 헤더]
A열: id        (타임스탬프 기반 자동 생성, 문자열)
B열: name      (신청자 이름, 문자열)
C열: email     (이메일, 문자열)
D열: course    (신청 과정명, 문자열)
E열: status    (처리 상태: '대기' | '승인' | '반려' | '삭제됨')
F열: created_at (등록 시각, 'yyyy-MM-dd HH:mm' 형식 문자열)

[필요한 함수]
1. createApplicant(name, email, course) — 신규 레코드 추가, 생성된 id 반환
2. getAllApplicants() — 전체 레코드를 객체 배열로 반환
3. findById(id) — id로 단건 조회
4. filterByStatus(status) — 상태별 필터링
5. updateStatus(id, newStatus) — 상태값 변경
6. softDelete(id) — status를 '삭제됨'으로 변경

[요구사항]
- 각 함수마다 JSDoc 주석 달기
- 에러 발생 시 Logger.log()로 메시지 출력
- 반복문에서 deleteRow() 사용 시 역순 순회 적용
- ES6+ 문법 사용 (const/let, 화살표 함수, 템플릿 리터럴)
⚠️ 동시 접근 주의: 여러 사람이 동시에 폼을 제출하거나 스크립트가 같은 시트에 동시에 쓸 경우, 행이 겹치거나 데이터가 손상될 수 있어요. GAS에는 LockService라는 잠금 기능이 있어요. 동시 접근이 우려된다면 LockService.getScriptLock()으로 한 번에 하나의 실행만 허용하도록 설정하세요.
