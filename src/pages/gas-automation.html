2. 신청자 이메일로 신청 확인 이메일 발송
   - 제목: "[교육 신청 완료] {이름}님의 신청이 접수되었습니다"
   - 내용: 신청자 이름, 신청 과정, 담당자 연락처 (admin@company.com) 포함
3. 담당자(admin@company.com)에게 신청 알림 이메일 발송
   - 제목: "[신규 신청] {이름} — {신청과정}"
   - 내용: 전체 응답 내용 요약

[요구사항]
- e.namedValues를 사용해서 질문 이름으로 응답 접근
- 이메일 발송 실패 시 Logger.log()로 오류 기록

- 한국어 JSDoc 주석 포함
GAS — 시나리오 1 코드 구조 
/**
 * 구글 폼 제출 시 자동 실행되는 함수
 * 트리거 설정: 폼 연결 스프레드시트 → 트리거 → onFormSubmit
 * @param {Object} e - 폼 제출 이벤트 객체
 */
function onFormSubmit(e) {
  // namedValues: { '질문 텍스트': ['응답값'] } 형태
  const responses = e.namedValues;
  const name    = responses['이름'][0];
  const email   = responses['이메일'][0];
  const course  = responses['신청 과정'][0];
  const dept    = responses['소속 부서'][0];

  const adminEmail = 'admin@company.com';

  try {
    // ① 신청자에게 확인 이메일 발송
    MailApp.sendEmail({
      to: email,
      subject: `[교육 신청 완료] ${name}님의 신청이 접수되었습니다`,
      body: `안녕하세요, ${name}님!\n\n` +
            `아래 내용으로 신청이 완료되었습니다.\n\n` +
            `• 신청 과정: ${course}\n` +
            `• 소속 부서: ${dept}\n\n` +
            `문의사항은 ${adminEmail}으로 연락 주세요.\n\n감사합니다.`
    });

    // ② 담당자에게 알림 이메일 발송
    MailApp.sendEmail({
      to: adminEmail,
      subject: `[신규 신청] ${name} — ${course}`,
      body: `새로운 교육 신청이 접수되었습니다.\n\n` +
            `이름: ${name}\n이메일: ${email}\n` +
            `과정: ${course}\n부서: ${dept}`
    });

    Logger.log(`신청 처리 완료 — ${name} (${email})`);

  } catch (err) {
    Logger.log(`이메일 발송 실패: ${err.message}`);
  }
}

// ── 트리거 설정 방법 ──────────────────────────────
// Apps Script 에디터 → 왼쪽 메뉴 '트리거(⏰)' → '+ 트리거 추가'
// ① 실행할 함수: onFormSubmit
// ② 이벤트 소스: 스프레드시트에서
// ③ 이벤트 유형: 양식 제출 시
// ④ 저장 → 완료!

📬 시나리오 2: 조건부 Gmail 자동 발송 (템플릿 기반)

스프레드시트의 상태 컬럼 값에 따라 서로 다른 내용의 이메일을 자동 발송하는 패턴이에요. 예를 들어, 신청자 상태가 '승인'이면 승인 안내 메일을, '반려'이면 사유를 담은 안내 메일을 보내는 거죠. 매일 또는 매주 정해진 시각에 자동으로 체크·발송하도록 타임 트리거를 설정해요.

Claude Code 프롬프트 — 시나리오 2 
Google Apps Script 코드를 만들어줘.
스프레드시트의 상태 컬럼 값에 따라 조건부 이메일을 발송하는 함수야.

[스프레드시트 구조 — '신청자' 시트]
A열: id, B열: name, C열: email, D열: course
E열: status ('대기' | '승인' | '반려')
F열: notified ('Y' | 빈 값) — 이미 발송했는지 체크

[동작 로직]
1. '신청자' 시트에서 status가 '승인' 또는 '반려'이고
   notified 컬럼이 비어있는 행을 찾는다
2. 상태에 따라 다른 템플릿으로 이메일 발송:
   - 승인: "{{name}}님, {{course}} 과정 신청이 승인되었습니다. 교육 일정..."
   - 반려: "{{name}}님, 아쉽게도 이번 신청은 반려되었습니다. 사유: 정원 초과"
3. 발송 후 해당 행의 notified 컬럼에 'Y' 표시
4. 발송 건수를 Logger.log()로 기록

[요구사항]
- 함수 이름: sendStatusEmails()
- {{name}}, {{course}} 자리표시자를 실제 값으로 치환하는 replaceTemplate() 헬퍼 함수 포함
- 타임 트리거로 매일 오전 10시에 실행하도록 주석으로 설정 방법 안내
- 오류 시 Logger.log() 기록
GAS — 시나리오 2 핵심 코드 // 이메일 템플릿 정의 const EMAIL_TEMPLATES = { '승인': { subject: '[승인 완료] {{name}}님의 {{course}} 신청', body: `안녕하세요, {{name}}님!\n\n` + `{{course}} 과정 신청이 최종 승인되었습니다. 🎉\n\n` + `교육 시작 3일 전에 상세 일정 안내 메일을 별도 발송해드릴 예정이에요.\n\n` + `기대해 주세요! 감사합니다.` }, '반려': { subject: '[안내] {{name}}님의 {{course}} 신청 결과', body: `안녕하세요, {{name}}님.\n\n` + `아쉽게도 {{course}} 과정 신청이 이번 차수에는 반려되었습니다.\n` + `사유: 신청 인원 초과 (선착순 마감)\n\n` + `다음 차수에 다시 신청해 주시면 우선 검토해드릴게요.\n\n감사합니다.` } }; /** * 템플릿의 자리표시자를 실제 값으로 치환합니다. * @param {string} template - {{key}} 형태의 자리표시자가 있는 문자열 * @param {Object} data - { key: value } 치환 데이터 */ function replaceTemplate(template, data) { return Object.entries(data).reduce((str, [key, value]) => { // 정규식으로 {{key}} 패턴을 모두 치환 return str.replace(new RegExp(`{{${key}}}`, 'g'), value); }, template); } /** * 매일 오전 10시에 실행 — 미발송 상태 이메일 일괄 처리 * * 트리거 설정: * Apps Script → 트리거 → + 추가 * 실행함수: sendStatusEmails * 이벤트 소스: 시간 기반 * 시간 유형: 일별 타이머 → 오전 9시~10시 */ function sendStatusEmails() { const sheet = SpreadsheetApp.getActiveSpreadsheet() .getSheetByName('신청자'); const data = sheet.getDataRange().getValues(); let sentCount = 0; // i=1부터: 1행은 헤더 for (let i = 1; i < data.length; i++) { const [id, name, email, course, status, notified] = data[i]; // 발송 대상: 승인 또는 반려이면서 아직 미발송 if ((status === '승인' || status === '반려') && notified !== 'Y') { const tmpl = EMAIL_TEMPLATES[status]; const vars = { name, course }; try { MailApp.sendEmail({ to: email, subject: replaceTemplate(tmpl.subject, vars), body: replaceTemplate(tmpl.body, vars) }); // F열(6번째)에 'Y' 표시 sheet.getRange(i + 1, 6).setValue('Y'); sentCount++; } catch (err) { Logger.log(`발송 실패 — ${name} (${email}): ${err.message}`); } } } Logger.log(`발송 완료: ${sentCount}건`); }

📅 시나리오 3: 스프레드시트 데이터 → Google Calendar 일정 자동 생성

일정 데이터가 스프레드시트에 있다면, 매번 Calendar에 직접 입력하지 않아도 돼요. GAS가 시트를 읽어 자동으로 구글 캘린더 일정을 만들어줍니다. 팀 교육 스케줄, 프로젝트 마일스톤, 행사 일정 관리에 딱 맞아요. 중복 생성 방지 로직도 함께 넣어서 안전하게 운영할 수 있어요.

Claude Code 프롬프트 — 시나리오 3 
Google Apps Script 코드를 만들어줘.
스프레드시트의 일정 데이터를 읽어 Google Calendar에 자동으로 일정을 생성하는 함수야.

[스프레드시트 구조 — '교육일정' 시트]
A열: event_title (일정 제목, 예: "AI 기초반 1회차")
B열: start_date  (시작일, 예: "2026-04-07")
C열: start_time  (시작 시각, 예: "14:00")
D열: end_time    (종료 시각, 예: "17:00")
E열: location    (장소, 예: "회의실 A")
F열: description (설명, 선택사항)
G열: calendar_event_id (생성된 이벤트 ID 저장 — 중복 방지용, 처음엔 빈 값)

[동작 로직]
1. '교육일정' 시트의 모든 행을 읽는다
2. G열(calendar_event_id)이 비어있는 행만 처리 (이미 생성된 건 스킵)
3. 시작/종료 Date 객체를 조합하여 CalendarApp으로 일정 생성
4. 생성된 이벤트 ID를 G열에 저장 (다음 실행 시 중복 생성 방지)
5. 생성된 건수 Logger.log() 기록

[요구사항]
- 함수 이름: syncCalendarEvents()
- CalendarApp.getDefaultCalendar() 사용
- 날짜/시간 파싱 헬퍼 함수 포함
- 중복 방지 설명 주석 포함
- 일정 생성 실패 시 해당 행 스킵 후 계속 진행
GAS — 시나리오 3 핵심 코드 
/**
 * 날짜 문자열과 시각 문자열을 Date 객체로 조합합니다.
 * @param {string} dateStr - "2026-04-07" 형식
 * @param {string} timeStr - "14:00" 형식
 * @returns {Date}
 */
function parseDateTime(dateStr, timeStr) {
  const [year, month, day] = dateStr.split('-').map(Number);
  const [hour, minute]     = timeStr.split(':').map(Number);
  // month는 0-based이므로 -1
  return new Date(year, month - 1, day, hour, minute);
}

/**
 * 스프레드시트의 교육 일정을 Google Calendar에 동기화합니다.
 * G열에 이벤트 ID가 있는 행은 이미 생성된 것이므로 스킵해요.
 *
 * 트리거: 수동 실행 또는 일별 타이머로 설정 가능
 */
function syncCalendarEvents() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName('교육일정');
  const data    = sheet.getDataRange().getValues();
  const calendar = CalendarApp.getDefaultCalendar();

  let createdCount = 0;

  for (let i = 1; i < data.length; i++) {
    const [title, startDate, startTime, endTime,
           location, description, existingId] = data[i];

    // G열에 이미 이벤트 ID가 있으면 중복 생성 방지 — 스킵
    if (existingId) {
      Logger.log(`스킵 (이미 생성됨): ${title}`);
      continue;
    }

    // 빈 행 스킵
    if (!title || !startDate || !startTime) continue;

    try {
      const startDt = parseDateTime(startDate.toString(), startTime.toString());
      const endDt   = parseDateTime(startDate.toString(), endTime.toString());

      // 캘린더 일정 생성
      const event = calendar.createEvent(title, startDt, endDt, {
        location:    location    || '',
        description: description || ''
      });

      // 생성된 이벤트 ID를 G열에 저장 (중복 방지)
      sheet.getRange(i + 1, 7).setValue(event.getId());
      createdCount++;

      Logger.log(`일정 생성 완료: ${title} (${startDate} ${startTime}~${endTime})`);

    } catch (err) {
      Logger.log(`일정 생성 실패 — ${title}: ${err.message}`);
      // 실패해도 다음 행 계속 처리
    }
  }

  Logger.log(`총 ${createdCount}건의 일정 생성 완료`);
}

🔄 중복 방지 전략이 왜 중요한가요?

트리거를 매일 실행하면, 이미 생성된 일정을 또 만들 위험이 있어요. 이를 막으려면 "이미 처리했음"을 시트에 표시해야 해요. 위 코드처럼 생성된 이벤트 ID를 G열에 저장하면, 다음 실행 시 해당 행을 건너뛰어요. 나중에 일정을 수정해야 한다면, G열 값을 지우고 다시 실행하면 돼요.
