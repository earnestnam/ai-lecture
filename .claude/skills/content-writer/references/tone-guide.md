# 비개발자 대상 글쓰기 가이드

## 핵심 원칙

이 사이트의 독자는 **개발 경험이 없는 일반 직장인**이다.
마케터, 기획자, 운영팀, 회계팀 등 AI를 처음 업무에 도입하려는 사람들이다.

---

## 톤 & 문체

### 기본 문체
- **"~합니다" 체** 사용 (격식체이되 딱딱하지 않게)
- 친근하고 실용적인 어투
- 독자를 "여러분"으로 호칭

### 좋은 예 vs 나쁜 예

| 나쁜 예 | 좋은 예 |
|---------|---------|
| "API 엔드포인트에 파라미터를 전달하십시오" | "버튼을 클릭해서 결과를 받아오세요" |
| "LLM의 토큰 리밋을 고려해야 합니다" | "AI에게 한 번에 너무 많은 내용을 보내면 잘릴 수 있어요" |
| "프롬프트 엔지니어링 기법을 적용하십시오" | "AI에게 더 잘 부탁하는 방법을 알아볼게요" |
| "멀티모달 입력을 지원합니다" | "글뿐 아니라 사진, 파일도 함께 보낼 수 있어요" |

## 기술 용어 처리 규칙

1. **첫 등장 시** 반드시 괄호로 쉬운 설명 추가
   - 예: "LLM(대규모 언어 모델 — 방대한 텍스트로 학습한 AI)"
2. **반복 사용 시** 쉬운 말로 대체하거나 그냥 "AI"로 통일
3. **금지 표현** (설명 없이 단독 사용):
   - API, 엔드포인트, 파라미터, 토큰, 배포, 스크립트, 레포지토리

## 콘텐츠 구조 패턴

### 도입부 템플릿
```
[공감 문장 — 독자의 현재 상황 묘사]
[문제 제기 — 왜 이게 필요한가]
[이 페이지에서 배울 것 — 구체적 약속]
```

예:
> "혹시 ChatGPT를 써봤는데 기대만큼 안 됐다는 느낌, 받아보셨나요?
> 그건 AI가 부족한 게 아니라, 부탁하는 방법이 달랐기 때문일 가능성이 높아요.
> 이 페이지에서는 AI가 바로 원하는 결과를 내줄 수 있도록 '잘 부탁하는 법'을 알아봅니다."

### 팁 박스 형식
```html
<div class="tip-box">
  <strong>💡 팁:</strong> [실용적인 한 줄 팁]
</div>
```

### 경고 박스 형식
```html
<div class="warning-box">
  <strong>⚠️ 주의:</strong> [놓치기 쉬운 함정]
</div>
```

### 스크린샷 자리표시자
```html
<div class="screenshot-placeholder">
  📸 스크린샷: [화면 설명 — 무엇을 캡처해야 하는지]
</div>
```

## 예시 선택 기준

독자의 업무 맥락과 가까운 예시를 우선한다:

**좋은 예시 소재**:
- 이메일 초안 작성
- 회의 요약
- 보고서·기획서 작성
- 데이터 정리 (Excel)
- 발표 자료 아이디어
- 고객 문의 답변

**피해야 할 예시 소재**:
- 코드 작성 (개발자용 예시)
- 학술 논문 작성
- 너무 전문적인 특정 산업 예시 (법률, 의학 등)

## 체크리스트 작성 규칙

체크리스트는 **행동 중심**으로 작성한다:

```html
<div class="checklist">
  <h3>✅ 실습 체크리스트</h3>
  <label><input type="checkbox" data-key="ai-why-1"> [동사]로 시작하는 구체적 행동</label>
  <label><input type="checkbox" data-key="ai-why-2"> [동사]로 시작하는 구체적 행동</label>
</div>
```

- `data-key` 값: `{페이지ID}-{순번}` 형식
- 체크 항목은 5개 이하로 제한
- "~이해하기" 금지 → "~해보기", "~확인하기" 등 행동형으로
